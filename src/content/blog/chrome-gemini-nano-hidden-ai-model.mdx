## Maliyet Karşılaştırması: Yerel vs API

AI inference maliyetleri, model ölçeğine, sağlayıcıya ve kullanım hacmine göre değişiyor. İşte güncel OpenRouter fiyatlarıyla kıyaslama:

| Model | Tip | Maliyet (1M token) | Not |
|-------|-----|-------------------|-----|
| Gemini 2.0 Flash (API) | Cloud | 0.10$ input / 0.40$ output | En popüler ucuz model |
| Gemini 1.5 Flash (API) | Cloud | 0.075$ input / 0.30$ output | Hâlâ güçlü |
| GPT-4o Mini (API) | Cloud | 0.15$ input / 0.60$ output | OpenAI'ın ucuz seçeneği |
| **Gemini Nano (yerel)** | On-device | **0$ token ücreti** | Tek seferlik cihaz maliyeti |
| Llama 3.2 3B (yerel) | On-device | 0$ token ücreti | Ücretsiz, açık kaynak |

Gemini Nano'nun token başına maliyeti sıfır. Ancak bu "bedava" değil — cihazın GPU/RAM kapasitesine yatırım yapmanız gerekiyor. Ortalama bir AI PC'nin ek maliyeti 500-1500$ arasında.

**Gerçekçi bir örnek:**

Bir SaaS uygulaması günde 10.000 API çağrısı yapıyor. Her çağrı 1000 token input, 500 token output:

- **API ile aylık maliyet:** 10.000 × 30 × (0.10$ × 0.001 + 0.40$ × 0.0005) = **195$/ay**
- **Yerel Gemini Nano ile:** Cihaz maliyeti ~800$, sonra sıfır token ücreti. 5. ayda API'den daha ucuz.

## Chrome Prompt API: Web Siteleri Yerel AI Kullanabilir

Chrome Prompt API, web geliştiricileri için yeni bir pencere açıyor. Bu API sayesinde web siteleri, kullanıcının cihazındaki yerel modeli kullanabilir — kullanıcının izniyle.

```javascript
// Chrome Prompt API kullanım örneği
const model = await window.ai.languageModel.create({
  systemPrompt: "Sen bir finans asistanısın."
});

const response = await model.prompt("2026 yılında Bitcoin fiyatı ne olur?");
console.log(response);
```

**Chrome Prompt API avantajları:**

1. **Gizlilik:** Veriler cihazda kalır, üçüncü taraflarla paylaşılmaz
2. **Hız:** Sunucu round-trip'i yok, gecikme 50-200ms
3. **Maliyet:** Geliştirici için sıfır API ücreti
4. **Çevrimdışı:** İnternet olmadan da çalışır

Ancak dikkat edilmesi gereken nokta: web sitesi, kullanıcının cihazındaki modele erişim istediğinde kullanıcı onay vermeli. Chrome bu onayı yönetir.

according to [Chrome AI Blog](https://developer.chrome.com/docs/ai/prompt-api) (2026)

## Kimler Yerel AI Kullanmalı?

Yerel AI her durumda API'den daha iyi değil. Doğru kullanım senaryolarını belirlemek önemli:

**Yerel AI için ideal:**

- Sık tekrarlanan, basit sorgular (刻, tokenize, özetle)
- Gizlilik kritik uygulamalar (tıbbi, finansal veriler)
- Düşük bağlantılı ortamlar (uçak, seyahat)
- Yüksek hacimli, düşük değerli işlemler (log analizi, spam filtreleme)

**API için ideal:**

- Karmaşık muhakeme görevleri (bilim, hukuk, tıp)
- Güncel bilgi gerektiren sorgular
- Çok dilli veya kültürel bağlam gerektiren içerik
- Büyük model kapasitesi gerektiren görevler

## WebGPU: Browser'da AI'ın Temel Taşı

Chrome'un yerel AI'ı çalıştırabilmesinin arkasındaki teknoloji WebGPU. WebGPU, tarayıcılarda GPU'ya doğrudan erişim sağlayan bir API — WebGL'nin modern halefi. WebGPU olmadan, tarayıcıda AI modeli çalıştırmak ya imkansız ya da aşırı yavaş.

**WebGPU destekleyen tarayıcılar:**
- Chrome 113+ (varsayılan açık)
- Edge 113+
- Firefox 130+ (deneysel)
- Safari 17.4+

WebGPU'nun AI kullanımı sadece Gemini Nano ile sınırlı değil. Stability AI, DeepSeek ve Mistral modelleri de WebGPU üzerinde çalışabilir. Bu, "tarayıcıda AI" ekosistemini genişletir.

```javascript
// WebGPU ile model yükleme örneği
const adapter = await navigator.gpu.requestAdapter();
const device = await adapter.requestDevice();

// Modeli WebGPU üzerinde çalıştır
const encoder = device.createCommandEncoder();
```

cite: [WebGPU W3C Specification](https://www.w3.org/TR/webgpu/) (2026)
cite: [Chrome WebGPU Blog](https://developer.chrome.com/docs/webgpu) (2026)
